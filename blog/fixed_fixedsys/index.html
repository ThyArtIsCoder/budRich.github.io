23 Nov 2017 22:35 | budRich | fonts fixedsys rice fontforge

I’m not sure why, but Fixedsys is one of my all time favorite fonts. I particularly like it as a monospaced terminal and programming font (Fixedsys is the font used on this site to display code and ASCII art). But it hasn’t been painless to get the terminal to accept Fixedsys.

The problem from the start is that the available version of Fixedsys is a .ttf font. After some digging around the Internet, I found this article on askubuntu.com.

I followed the instructions and managed to get the font in the terminal.

but

The font wasn’t recognized in .Xresources. I had to put this dirthack in my .bashrc:

printf '\e]710;%s\007' "xft:Fixedsys Excelsior 3.01\-L2 Mono:style=Regular"

Sure, it worked, but the hack became the cause of some strange, undesired behavior.

But I couldn’t give up, not when I was so close. So I did some experimenting, and I think the issue of .Xresources not recognizing the font, was the font name. So following the instructions in the askubuntu.com article, but renaming the font to FixedFixedsys did the trick!

Below is a summary of the askubuntu.com article with some added screenshots and the name change. I wanted to save it here, and i guess the same technique can be used to enable other fonts.

Before anything else, make sure fontforge is installed. And get this modified version of Fixedsys

Open FSEX301-L2.ttf with fontforge.

fontforge1.png

In the dialog asking to load bitmap fonts, select: No

Open the menu File → Execute Script... (Ctrl+.) and paste the following code in the input box:

SelectAll()
foreach
   if (GlyphInfo("Width") != 80)
      DetachAndRemoveGlyphs()
   else
      b = GlyphInfo("BBox")
      if (b[0] < 0 || b[1] < -30 || b[2] > 80 || b[3] > 130)
         DetachAndRemoveGlyphs()
      endif
   endif
endloop
SelectNone()

Select FF and click OK (it will take a while, you will have to wait until the Execute Script window disappears)

fontforge2.png

Next open Element → Font Info... menu In PS Names change the Fontname, Family Name, and Name for Humans to FixedFixedsys.

fontforge3.png

In the same Font Information window, select OS/2. In the Misc. tab change the PFM Family to Monospace.

fontforge4.png

In the Metrics tab, set Win Descent Offset and HHead Descent Offset both to 0 (*).

fontforge5.png

In the Panose tab, change Proportion to Monospaced.

fontforge6.png

In the same Font Information window, select Lookups. In the GSUB tab, and in the GPOS tab, select each line in turn and click Delete.

fontforge7.png

In the same Font Information window, click OK, and in the Change UniqueID? window that appears click Change.

fontforge8.png

Open the Element → Bitmap Strikes Available... (Ctrl+Shift+B) menu. In the window that appears set Pixel Sizes to 16 and click OK.

fontforge9.png

Select File → Generate Fonts... menu (Ctrl+Shift+G), choose a different directory and/or a different filename if needed, make sure the output format is TrueType, and click Generate. In the Non-standard Em-Size window that appears click Yes.

fontforgeA.png

Also click Generate in the next window that appears.

fontforgeB.png

Quit FontForge File → Quit (Ctrl+Q). In the Font changed window that appears click Don't Save.

Move the generated file to the ~/.fonts directory (create it if it doesn’t exist).

Now you should be able to set the font in .Xresources .

I would also suggest that you add the following rule to ~/.fonts.conf to disable boldness from the font. (it will render weirdly otherwise, and Fixedsys is, imo, bold enough as it is.)

<match target="pattern">
    <test qual="any" name="family">
        <string>FixedFixedsys</string>
    </test>
    <test name="weight" compare="more">
        <const>medium</const>
    </test>
    <edit name="weight" mode="assign" binding="same">
        <const>medium</const>
    </edit>
</match>
