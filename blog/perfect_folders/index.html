I have tried many different ways to organize my files. I think the current setup I have is by far the best. It uses Dropbox to ensure cloud based backup and Hugo to create this site you are looking at where many of my actual dotfiles and scripts are viewable and auto-updated.

folder structure

This might look overcomplicated and hard to setup, but it actually makes a lot of sense and takes no time to setup.

aliases:
box = Dropbox folder, $HOME/box/Dropbox
S  = symbolic link
H  = hard link
D  = normal folder
F  = normal file
P  = original path

home folder

S bin # P=box/bin, included in PATH.
D box # contains Dropboxfolder
S dox # P=box/box/dox
D dwn # downloads folder
  D w # downloads from browser
  D s # downloaded series
  D m # downloaded music
  D b # downloaded books
  D f # downloaded movies
  D y # downloaded youtube videos
  D p # downloaded podcasts
  D v # downloaded musicvideos
S etc # P=box/box/etc
D dot # dotfiles
  H .bashrc # P=box/www/static/dots/bashrc/files/bashrc
  D .config
    D i3
      H config # P=box/www/static/dots/config/i3/config/files/config
S new # P=box/box/new
D pix # pictures folder
  S pwp # P=box/box/pix/pwp
  S gif # P=box/box/pix/gif
  S wp  # P=box/box/pix/wp
  S ss  # P=box/box/pix/ss
  S r   # P=box/box/pix/r
  D tmp
D pub # public folder
D snd # sound folder
  D muz # music folder
    S dwn # P=~/dwn/m
    D loc # local music
  D tlk # talk(pod) folder
    S dwn # P=box/~/dwn/p
  S bud # P=box/box/snd/bud
D src # included in PATH
  H igi   # P=box/www/static/scripts/igi/files/igi
  H i3run # P=box/www/static/i3ass/i3ass/files/i3ass
D tmp # Desktop folder
D vid # video folder
  S s # P=~/dwn/s
  S f # P=~/dwn/f
  S y # P=~/dwn/y
  S v # P=~/dwn/v
  S bud # P=box/box/vid/bud
S www    # P=box/box/www
S .fonts # P=box/hem/.fonts
D .config
  D i3
    H config # P=box/www/static/dots/config/i3/config/files/config
  S sublime-text-3 # P=box/hem/.config/sublime-text-3
  F user-dirs.dir
H .bashrc # P=box/www/static/dots/bashrc/files/bashrc
F .gtkrc-2.0
D .gnome2

www folder

I am not showing the whole actual folder structure here. Only how .bashrc is stored. I think you can guess how the rest looks. There is also alot of site specific html/image/sass files in this folder that is not of importance for this article.

D content
  D dots
    F bashrc.md
D static
  D dots
    D bashrc
      F banner.html   # the ASCII banner
      F barquote.html # the sidebar quote
      D files
        F bashrc # this is the actual bashrc file

The bashrc.md file consists of two parts:
A front matter (in YAML format) that holds info like date and author. I also added a custom value to the front matter called fspath . This holds the path to the file on the filesystem, in this case: ~/.bashrc. The rest of the file is markdown that will be the content on the site between the ASCII banner and the file.

setup

The easiest way to create linked folders is to open two GUI file manager windows. Almost every file manager will create a linked folder/file if you hold Ctrl+Shift when dropping the original folder where you want the link (This even works on windows). You can rename and move the links but if you rename or move the originals you need to create new links.

Creating the links from the www folder is another story and I have made two scripts to help with this:

subact

I use subact to “publish” a script or a dot file. It analyzes the window title of Sublime Text which contains the path to the currently open file. The script moves this file to the right place in box/www/static and creates banner.html and barquote.html. It then create a hard link back to it’s original place. It also creates the corresponding markdown file in box/www/content and puts the original path in the front matters fspath variable. It then opens the markdown file in Sublime. I have the script hooked up to a hotkey in my i3 config, so whenever I edit a file I want to publish I fire Mod4+a and it’s done. It also takes a list of files as an argument, if given it use the list instead of the title of sublime.

dotcrawler

This is the opposite of subact. It crawls through box/www/content and creates links according to fspath on the filesystem to the files in box/www/static. I run this script on a newly installed distro.

Now I can edit the files in their normal location (f.i. ~/.bashrc) but since this is a hardlink to the file ~/box/Dropbox/www/static/dots/bashrc/files/bashrc the changes will be saved there. Synched to Dropbox automatically and seen on the homepage when i rebuild it with Hugo.

I have disabled show hidden files in my file manager, so I can browse the filesystem without triggering my OCD!
