These are some of my personal preferences and thoughts on how to configure i3. Read the official userguide often to see what options you got. I have left out a lot of functionality in my config that i don’t use.

Your config file can be located at different places depending on how you installed i3.

variables

the first line of the default config file looks like this:

set $mod Mod4

It creates a variable called $mod, that contains the value Mod4. Variables are your friends. Now you don’t have to remember which Mod you have set and can create key bindings/hotkeys/shortcuts like this:

bindsym $mod+Return exec --no-startup-id exec urxvtc

But variables can be even more powerful. I have this at the beginning of my configfile:

set $win bindsym Mod4
set $exe exec --no-startup-id exec

With these variables, the same binding can be written like this:

$win+Return $exe urxvtc

You will most likely make a lot of similar program-launching-key-bindings. With this method i think the config becomes easier to both read and write.

Below are all my set variables. (Some of them are dependent on external applications, but you get the idea.)

set $exe exec --no-startup-id exec

set $left1   y
set $down1   u
set $up1     i
set $right1  o

set $left2   h
set $down2   j
set $up2     k
set $right2  l

set $left3   n
set $down3   m
set $up3     comma
set $right3  period

set $fntsz 8
set $browser -c Vivaldi-stable -e vivaldi
set $editor -i sublime_text -e subl
set $filemanager -i spacefm -e spacefm 
set $bigsize   10 px or 10 ppt
set $smallsize 3 px or 3 ppt
set $bigmove   30 px
set $smallmove 1 px

set $alt Mod1
set $win bindsym Mod4
set $fdmtm floating disable, move to mark

set_from_resource $act i3wm.act #111313
set_from_resource $ina i3wm.ina #111313
set_from_resource $red i3wm.red #FF0000
set_from_resource $fnt i3wm.fnt Terminus

The last three variable declarations look different. That’s because they are read from .Xresources, it is perfect for colors since you can set colors for all terminals and many (i3, polybar, rofi e.t.c.) other programs in one file.

The rest of your config file will be hotkeys, settings, window rules, mode definitions, bar configuration and programs you want to autostart with i3.

Nothing is mandatory and you can include or exclude whatever you want. Personally i don’t use modes or the bar so they are not included in my configfile.

hotkeys

The number of hotkeys can become quite large over time. Try to have some sort of system to organize them. I have divided my bindings in to different categories:

# panicbuttons
# ============
this is where i put reload, exit, lock, shutdown e.t.c
Most of these bindings are complex on purpose, so i dont
use them by mistake.

# terminals
# =========
i have some different hotkeys to open different kinds of terminals.
All of these bindings are modifier(s)+Return

# i3run - sane applaunching for the modern tiler
# ==============================================
i3run(link) is a script i wrote to help launching programs with i3. All hotkeys in this category uses i3run and opens a new program in a new window. Most of them are $win+[NUMBER].

Switching apps is, in my case, the most used hotkeys, and i find it easiest to remember the programs when they are bound to a number. First i used $win+f firefox , $win+s sublime, and so on. But i found this being confusing and inconvenient. Its hard to remeber which letters that are free when adding new hotkeys, and it becomes ugly if you switch browser from firefox to chrome for example. So having the browser on $win+2 and a variable, $browser, set to my choise at the top of the script makes more sense to me. 

# barmenu
# =======
here i put bindings that launches menus or manipulates the bar (toggling the bar or the tray.) It made sense to combine these two since most of my menus are dmenus that appears over the bar when launched.

# media
# =====
bindings to control music, volume e.t.c

# upload content
# ==============
here is hotkeys to make screenshots/casts or create 'pastes' of selected text.

# wallpapermanagement
# ===================
keys to manipulate the wallpaper

# mousekeys
# =========
keys related to the mouse

# debugging
# =========
dumping the layout, and getting other debuginfo

# moving the size of the focus
# ============================
keys to move, resize or focus windows. I have this logic:
H,Y,N = Left ; shrink width 
J,U,M = Down ; shrink height
K,I,(comma) = Right ; grow width
L,O,(period) = Up ; grow height

It's inspired by the layout of the arrowkeys and the fact that 'Up' is the least used direction. I put resize action on the home row, because it is the most used action. Focusing is 9/10 times made by appspecific hotkey bound to i3run.

# workspace stuff
# ===============
I only have 3 workspace hotkeys set, because i seldom use more then one workspace. This is also the reason why I have given the default workspace hotkeys to appswitching, and use $win+F1/F2/F3/F12 for the workspaces.

# misc
# ====
Other hotkeys.

You can write comments in your config by prefixing the comment with a hash #.

appearance

Here are some thoughts on configuring the appearance. I have a script that displays the window title in my polybar, so I doesn’t really need the titlebar. But it can be nice to have a titlebar on floating windows, so you can drag them without holding down Mod1.

To keep things as minimal as possible, I have set the text color to the same as the border color. This method might however affect the looks of your bar in a negative way. And I recommend that you use Window title format instead.

By default I have all my windows appear floating, and then I define windows that i want to be tiled (and moved to a mark, $fdmtm (I may or may not explain this further in the future)).

# appearance
# ==========
for_window [floating] border normal 1
for_window [tiling] border pixel 1

# widow Colors           border  bg     text    indic
client.focused           $act    $act   $act    $act
client.focused_inactive  $ina    $ina   $ina    $ina
client.unfocused         $ina    $ina   $ina    $ina
client.urgent            $red    $red   $red    $red

# window rules
# ============
for_window [class="^.*"] floating enable

for_window [class="Vivaldi-stable"]  $fdmtm master
for_window [instance="irssi"]        $fdmtm sidemain
for_window [instance="ncmpcpp"]      $fdmtm sidetop

for_window [instance="floatterm"] resize set 300 px 150 px

no_focus   [instance="keydisplay"]
for_window [instance="keydisplay"] sticky enable, border none

for_window [window_type="dialog"]  floating enable
for_window [window_type="splash"]  floating enable
for_window [window_type="utility"] floating enable

At the bottom of my config I have defined programs I want to autoexecute with i3. Which at the moment is none, I have put that stuff in .xinitrc instead.
